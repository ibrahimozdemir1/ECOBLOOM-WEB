Project Details

1. Requirements

1.1. Functional Requirements

  1. Data Collection:
    • The system must retrieve weather and environmental data specific to the user's location.
    • The weather data should be updated regularly to ensure the accuracy of recommendations.
  2. Prediction and Analysis:
    • Machine learning algorithms should analyse the gathered data to forecast future temperature.
    • The system must evaluate how temperature influences different plant species to provide personalized recommendations.
    • All collected and processed data must be stored in a structured database for future analysis and system improvement.
  3. Plant Recommendations: The system must suggest plant or crop species based on users' local or specified temperature data.
  4. Interactive Support:
    • A chatbot should provide users with detailed plant care guidance.
    • The chatbot must also simulate purchase options by suggesting reliable sources for plants.
  5. User Interaction: The web interface must allow users to input their location or temperature interval for plant recommendations.
  6. Integration:
    • The system must integrate with external APIs and maintain secure connections for data exchange.
    • The database should handle plant and weather information, ensuring seamless access and updates.

1.2. Non-Functional Requirements

  1. Performance:
    • The system must respond to user queries within five seconds under normal conditions.
  2. Reliability:
    • The system should ensure high availability with minimal downtime.
    • It must include error-handling mechanisms to notify users of issues and retry failed processes.
  3. Usability:
    • The user interface should be intuitive and accessible to both novice and experienced users.
    • A help section or tutorial must be available to guide users through the system's features.
  4. Security:
    • User data should be encrypted both during transmission and when stored in the database to prevent unauthorized access.
  5. Maintainability:
    • The system should adopt a layered architecture, enabling easy updates and maintenance without impacting other components.
    • Comprehensive documentation must support future development and troubleshooting.
  6. Portability:
    • The web application must be compatible with popular browsers, including Chrome, Edge, and Safari, to ensure broad accessibility.

3. Design

ECOBLOOM follows a Layered Architecture integrated with Model-View-Controller (MVC) principles:

Layers

  • Presentation Layer: Manages user interaction.
  • Application Layer: Handles business logic and machine learning models.
  • Data Layer: Operates a SQL-based database for users, plants, and climate data.
  • External Service Layer: Interfaces with the external systems.

Main Components

  • User Interface: Captures user interaction.
  • Machine Learning Module: Predicts future temperature ranges using linear regression.
  • PlantRecommendation Module: Matches weather predictions to plant climate tolerances.
  • Chatbot Module: SVM-based intent analysis, care and purchase guidance.
  • Database Module: Secure, structured storage.

Diagrams and Specifications

Use case, class, activity, and admin flow diagrams were created and refined throughout the design phase. Each class was specified in detail with attributes, methods, and interaction logic.

4. Implementation and Testing

Technology Stack

  • Language/Frameworks: Python 3.10, Flask, SQL, scikit-learn
  • Database: SQL 8.0+
  • Tools: GitHub (version control), Swagger (API testing), Selenium (UI testing), PyCharm / VSCode

Implementation Phases

  • COMP4910 (Fall): Requirement analysis, UML design, architectural decisions
  • COMP4920 (Spring): Full-stack implementation, UI/frontend development, chatbot and ML model integration, system testing

Testing Activities

  • Unit Tests: WeatherAPI, ML prediction, plant filter logic
  • Integration Tests: Data flow from input to recommendation and chatbot
  • Security Tests: Verified encryption and access control on stored data

All critical components were tested and debugged collaboratively, ensuring a stable system.

5. Conclusions

Summary

ECOBLOOM successfully delivers a robust and user-friendly web application that personalizes plant and crop recommendations based on environmental data. The completed system supports:

  • Automatic and manual weather input
  • Weather prediction via ML
  • Care and purchase support via chatbot
  • Sorting and filtering of recommendations

Benefits

  • To Users: Provides personalized, data-driven plant recommendations and care guidance, empowering both amateur gardeners and experienced farmers to achieve optimal results.
  • To Plants: Promotes healthy growth by recommending species well-suited to specific climates and providing detailed care instructions.
  • To Nature: Encourages sustainable gardening and farming practices, reducing resource waste and fostering environmental balance.
  • To Society: Enhances agricultural productivity and contributes to food security by assisting farmers in selecting appropriate crops.

Future Work

While ECOBLOOM has met its core objectives, there are several promising directions for future development:

  1. Additional Functionality
    • Integration with more APIs for advanced weather and soil data
    • A mobile app version to enhance accessibility
    • Expanded chatbot capabilities with voice interaction and multilingual support
  2. Performance Improvements
    • Support for distributed systems to handle larger user bases
    • Advanced optimization techniques for faster recommendation generation
  3. Expanded Data
    • Enrich the plant database with rare species and extreme climate adaptations
    • Include pest and disease data for preventive care recommendations
    • Provide comprehensive care advice for all listed plants
  4. Advanced Features
    • Real-time adaptive ML models based on user feedback and usage patterns
